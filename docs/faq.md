# 사용자 FAQ
- 마지막 업데이트: 2025-09-25
- 상태: 초안

## TL;DR
- 탐색 기본 순서: 루트 `README.md` → 폴더별 `README` → 세부 문서 TODO.
- 문서 상단 메타데이터(마지막 업데이트·상태)와 TODO를 먼저 확인해 최신성을 파악합니다.
- 새 노트 작성 시 템플릿을 복사하고 관련 인덱스를 반드시 갱신합니다.

## Q1. 어떤 문서부터 읽어야 하나요?
- 루트 `README.md`의 "구조 한눈에 보기" 표에서 관심 주제 폴더를 고릅니다.
- 해당 폴더의 `README`에서 대표 문서와 상태를 확인한 뒤 우선순위를 정합니다.
- 시간이 없다면 `docs/getting_started.md`의 [첫 5분 큐레이션](docs/getting_started.md#첫-5분-큐레이션)과 [사용자 유형별 바로가기](docs/getting_started.md#사용자-유형별-바로가기)를 참조하세요.

## Q2. 문서 상단의 "상태"는 무엇인가요?
- **초안**: 개요/링크 위주로 구성된 초기 버전으로, 주요 내용 보강이 필요합니다.
- **검토중**: 핵심 내용은 정리되어 있으며 예시·다이어그램 등 추가 개선을 기다립니다.
- **완료**: 현시점 기준으로 참고 가능한 상태입니다. 개선 아이디어가 있으면 TODO에 남겨 주세요.

## Q3. 새 노트를 추가하거나 수정하려면 어떻게 하나요?
- `docs/note_template.md`를 복사해 새 파일을 만들고 메타데이터를 채웁니다.
- 작성 가이드는 `docs/writing_guidelines.md`를 따르며, 명령어·경로는 코드 서식으로 표기합니다.
- 문서 수정 후 관련 README와 `docs/document_inventory.md`에 링크·상태를 업데이트합니다.
- 체크리스트가 필요하면 `docs/getting_started.md`의 [문서 보강자용 체크포인트](docs/getting_started.md#문서-보강자용-체크포인트)를 활용하세요.

## Q4. TODO나 보강 아이디어는 어디에 남기나요?
- 각 문서의 "TODO / 후속 연구" 섹션에 체크박스를 추가해 공유합니다.
- 폴더 단위 제안은 `docs/document_inventory.md`의 상태 메모 또는 우선순위 표에 기록합니다.
- 단기 작업 메모는 루트의 `today.md`나 `check.md`에 남겨도 좋습니다.
- 재방문 시 진행 상황을 빠르게 확인하려면 `docs/getting_started.md`의 [재방문 사용자를 위한 빠른 점검](docs/getting_started.md#재방문-사용자를-위한-빠른-점검)을 참고하세요.

## Q5. 빠르게 필요한 내용을 검색하려면?
- 루트에서 `rg <키워드>`를 실행해 전체 노트를 검색합니다.
- 특정 폴더만 검색하고 싶다면 `rg <키워드> <폴더경로>` 형태로 범위를 좁힙니다.
- 도구용 명령 예제가 필요하면 `tools_notes/` 폴더와 각 README의 "대표 문서 빠른 링크" 표를 확인하세요.
- 최근에 수정된 문서만 보고 싶다면 `git log --since="2 weeks ago" --name-only -- docs/` 명령으로 추적하세요.

## Q6. FAQ에 없는 질문은 어떻게 하나요?
- 관련 문서의 TODO에 질문을 기록하거나 `docs/document_inventory.md`의 유지관리 메모에 노트를 남깁니다.
- 공통 질문이라고 판단되면 이 문서를 업데이트하고 참조 링크를 추가하세요.

## Q7. 문서가 가이드라인을 따르는지 빠르게 확인하는 방법은?
- 각 문서 상단에 `마지막 업데이트`, `상태`, TL;DR 한 문장이 있는지 확인합니다.
- 섹션 구조가 **개요 → 핵심 개념 → 실무 포인트 → TODO → 참고 자료** 흐름을 따르는지 검토합니다.
- 명령어·경로가 인라인 백틱 또는 코드 블록으로 표시되어 있는지 확인하고, 누락되었다면 `docs/writing_guidelines.md` 체크리스트를 참고해 보강하세요.
- 개선 작업을 하기 전에 `docs/document_inventory.md`에서 동일 문서가 정비 예정인지 확인하면 충돌을 줄일 수 있습니다.

## Q8. 문서 인벤토리의 "사용자 권장 행동"은 어떻게 활용하나요?
- `docs/document_inventory.md`를 열어 담당하고 싶은 폴더의 권장 행동을 선택합니다.
- 해당 행동을 수행한 뒤, 결과나 추가 TODO를 동일 문서의 상태 메모나 우선순위 표에 갱신합니다.
- 작업 내용을 공유할 때는 관련 폴더 `README`의 대표 링크 표도 함께 업데이트해 다른 사용자에게 알리세요.
- 진행 현황을 기록할 때는 체크박스와 날짜를 함께 남기면 추후 추적이 쉬워집니다.

## Q9. 최근 업데이트를 빨리 확인하려면?
- `README.md`의 "진행 중 과제" 섹션과 `docs/document_inventory.md`의 "정비 우선순위 제안"을 먼저 확인하세요.
- 변경 이력이 궁금하다면 `git log -n 10 -- docs/` 또는 `git log --stat -- docs/getting_started.md`처럼 범위를 좁혀 확인합니다.
- 최신 문서 편집 흐름은 `docs/getting_started.md`의 [재방문 사용자를 위한 빠른 점검](docs/getting_started.md#재방문-사용자를-위한-빠른-점검)에서 체크리스트 형태로 확인할 수 있습니다.

## Q10. 다이어그램과 실습 로그는 어디에서 확인하나요?
- 다이어그램 원본은 `drawio/` 폴더에 있으며, 해당 문서의 "예제/다이어그램" 섹션에서 파일명을 직접 링크합니다.
- 이미지가 필요하면 `draw.io` 데스크톱 앱에서 `File > Export as > PNG`로 내보내고 문서에 `![설명](../drawio/<파일명>.png)` 형태로 삽입하세요. 내보낸 PNG는 `docs/assets/` 하위에 보관하는 것을 권장합니다.
- 최근 보강된 예시는 `basic_notes/algorithme/RAFT.md`(리더 선출 순서도), `tech/Bitlock.md`(TPM+PIN 흐름), `security_note/defend/NAC.md`(NAC 결정 트리), `tools_notes/docker/command.md`(실행 로그)에서 확인할 수 있습니다.
- 실행 로그는 개별 문서의 TL;DR 아래 또는 "실무/시험 포인트" 섹션에 추가되며, 실제 명령 결과를 삽입할 때는 블록 코드와 타임스탬프를 함께 기록해 주세요.
