- 문서 상단 메타데이터(마지막 업데이트·상태)로 최신성·완성도를 즉시 확인합니다.
- 작업 전 `docs/document_inventory.md`에서 정비 대상과 TODO를 체크해 중복을 줄입니다.
- 막히면 `docs/faq.md` 또는 폴더 README의 "대표 문서 빠른 링크" 표를 활용하세요.

## 첫 5분 큐레이션
1. `README.md`의 "구조 한눈에 보기" 표에서 오늘 살펴볼 폴더를 고릅니다.
2. 선택한 폴더의 `README`에서 ✅ 표시된(완료) 문서와 ⚙️/⏳ 표시된(보강 예정) 문서를 구분합니다.
3. 필요한 세부 문서를 열어 TL;DR·TODO를 확인하고 바로 필요한 정보부터 읽습니다.

> **TIP**: 시간이 5분밖에 없다면, TL;DR과 TODO만 훑고 바로 필요한 링크만 열어도 흐름을 파악할 수 있습니다.

## 저장소를 처음 접한 사용자라면
이 노트 모음은 보안·네트워크·운영체제·개발 주제를 아카이브한 개인 지식 창고입니다. 모르면 막히는 순서를 아래처럼 정해두면 길을 잃지 않습니다.

1. 루트 `README.md`의 "구조 한눈에 보기" 표로 폴더별 성격을 파악합니다.
2. 관심 있는 분야의 디렉터리를 열어 서브 문서 리스트를 확인합니다. 폴더 안에 `README` 또는 인덱스 문서가 있는 경우 먼저 읽어 방향을 잡습니다. `README`에는 "대표 문서 빠른 링크" 표가 있어 우선 확인할 파일과 상태를 한눈에 파악할 수 있습니다. (`security_note/README.md`, `basic_notes/README.md`, `tech/README.md`, `tools_notes/README.md` 등)
3. 문서 상단의 메타데이터(마지막 업데이트·상태)를 보고 최신성·완성도를 판단합니다.
4. 더 깊이 알아보고 싶다면 문서 끝의 참고 링크 또는 TODO에서 후속 자료를 확인합니다.

> **내비게이션 요령**: README에 없는 문서는 `rg <키워드>`로 바로 검색하고, 비슷한 이름의 문서가 많으면 `docs/document_inventory.md`의 상태 메모를 확인하세요.

## 문서 메타데이터 이해하기
문서는 상단에 다음 정보를 포함합니다.
```
- 마지막 업데이트: YYYY-MM-DD
- 상태: 초안/검토중/완료 (필요 시 부가 설명)
```
- **초안**: 개요나 링크만 있는 단계 – 주요 내용은 다른 문서나 외부 링크를 참고해야 합니다.
- **검토중**: 기반 내용은 갖춰졌으며, 예제·표·다이어그램 보강이 예정되어 있습니다.
- **완료**: 현재 기준으로 참고해도 좋은 상태. 향후 보강할 때는 README의 "진행 중 과제"를 갱신합니다.

문서 상태가 불명확하거나 오래된 경우 `docs/document_inventory.md`에서 점검 상태를 확인하고, 필요하면 TODO에 피드백을 남기세요.

## 검색과 탐색 팁

- 파일명은 가능한 한 주제를 그대로 드러내도록 작성되어 있으므로, 디렉터리 목록에서 키워드만으로도 문서를 추론할 수 있습니다.
- 동일 주제 문서가 여러 개 있을 경우 최신 문서를 `README`나 해당 폴더의 인덱스에서 링크합니다. 중복이 의심되면 `docs/document_inventory.md`에 메모를 추가하세요.
