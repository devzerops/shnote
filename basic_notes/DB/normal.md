# 개요
정규화(Normalization)는 릴레이션의 이상(Anomaly)을 제거하고 데이터 중복을 최소화하기 위해 함수 종속성을 분석해 테이블 구조를 재배치하는 과정입니다. 1NF부터 5NF까지 단계적으로 제약을 강화합니다.

# 핵심 개념
- **1NF**: 모든 속성이 원자값을 가지도록 반복 속성을 제거합니다.
- **2NF**: 1NF 만족 + 부분 함수 종속 제거(복합 기본키의 일부분에만 종속된 속성 분리).
- **3NF**: 2NF 만족 + 이행 함수 종속 제거(비키 속성이 다른 비키 속성에 종속 X).
- **BCNF**: 모든 결정자가 후보키가 되도록 강제, 3NF보다 엄격.
- **4NF**: 다치 종속(MVD)을 제거해 후보키 외 속성 간 다치 관계를 분리.
- **5NF**: 조인 종속을 제거해 릴레이션을 다시 결합해도 정보 손실이 없도록 보장.
- **비정규형**: 보고서 생성 등 성능을 위해 의도적으로 중복을 허용할 수도 있으며, 정규화/비정규화 균형이 중요.

# 실무/시험 포인트
- 업무상 조회 빈도가 높은 조인 구조는 과도한 정규화로 성능이 저하될 수 있으므로, 요약 테이블·인덱스 설계와 함께 고려해야 합니다.
- 정규화 수준 판단 시 **함수 종속도**를 먼저 분석해 결정자→종속자 관계를 명확히 기록합니다.
- 자격증(데이터베이스 기사, 정보처리기사)에서는 각 정규형 조건과 위반 사례, 정규화 절차 문제를 자주 출제합니다.
- OLAP/데이터 웨어하우스는 Star Schema처럼 비정규화된 구조를 채택하지만, 원장 시스템은 3NF 이상을 기본으로 합니다.
