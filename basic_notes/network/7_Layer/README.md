# 개요
OSI 7계층은 네트워크 통신 절차를 기능별로 분리하여 설계·분석·트러블슈팅을 쉽게 해 주는 참조 모델입니다. 계층별 장비와 데이터 단위(PDU)를 이해하면 원인 구간을 빠르게 좁힐 수 있습니다.

## 계층별 대표 장비·프로토콜
| 계층 | 데이터 단위 | 대표 장비/기능 | 주요 프로토콜 |
|------|--------------|----------------|----------------|
| L7 응용 | 데이터 | WAF, L7 스위치, 프록시 | HTTP, SMTP, DNS, FTP |
| L6 표현 | 데이터 | SSL/TLS 게이트웨이, 암호화 엔진 | TLS, X.509, JPEG |
| L5 세션 | 데이터 | 세션 브로커, RPC 게이트웨이 | NetBIOS, RPC, gRPC |
| L4 전송 | 세그먼트/데이터그램 | L4 스위치, FW(세션), 로드밸런서 | TCP, UDP, SCTP |
| L3 네트워크 | 패킷 | 라우터, L3 스위치, VPN 게이트웨이 | IPv4/IPv6, OSPF, IPSec |
| L2 데이터링크 | 프레임 | 스위치, 브리지, 무선 AP | Ethernet, 802.1Q, ARP |
| L1 물리 | 비트 | 케이블, 허브, 리피터, SFP | UTP, 광섬유, 전기 신호 |

## 브리지 vs 라우터
| 항목 | 브리지(L2) | 라우터(L3) |
|------|------------|-------------|
| 기준 주소 | MAC | IP |
| 테이블 종류 | MAC 주소 테이블 | 라우팅 테이블 |
| 브로드캐스트 처리 | 동일 브로드캐스트 도메인 | 기본적으로 차단, 필요 시 멀티캐스트 |
| 헤더 조작 | 프레임 헤더만 검사, 변경 불가 | 새 IP 헤더 생성 가능 |
| 용도 | 브로드캐스트 도메인을 나누지 않고 충돌 영역만 분리 | 서로 다른 네트워크를 연결하고 경로 선택 |

# 핵심 개념
- 계층은 **서비스(상위에게 제공)**와 **인터페이스(하위에게 의존)** 관계로 구성된다.
- 계층별 캡슐화/디캡슐화 과정에서 PDU 명칭이 달라지고 오류 위치를 구분하는 실마리가 된다.
- 실환경에서는 TCP/IP 4계층 모델을 더 많이 사용하지만, OSI 모델이 시험과 설계 문서의 공통 기준이다.

# 실무/시험 포인트
- 트러블슈팅 시 `L1 → L7` 순서로 확인(물리 → 링크 → 네트워크 → 전송 등)하면 빠르게 범위를 축소할 수 있다.
- L3 이상에서 암호화된 트래픽은 L7 장비에서 복호화(TLS termination) 절차를 마련해야 분석 가능하다.
- 자격증(정보통신기사, CCNA 등)에서는 계층별 PDU, 장비, 프로토콜 매칭 문제와 브리지/라우터 비교가 빈출된다.
