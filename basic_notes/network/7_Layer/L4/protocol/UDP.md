# 개요
UDP(User Datagram Protocol)는 비연결형·비신뢰성 전송을 제공하는 L4 프로토콜입니다. 최소한의 헤더(8바이트)만 사용해 지연을 줄이며, 스트리밍·DNS·VoIP처럼 빠른 응답이 필요한 서비스에 적합합니다.

## 헤더 구조
| 필드 | 크기 | 설명 |
|------|------|------|
| Source Port | 16bit | 송신 프로세스 식별 |
| Destination Port | 16bit | 수신 프로세스 식별 |
| Length | 16bit | 헤더+데이터 전체 길이 |
| Checksum | 16bit | 오류 검출 (IPv4 optional, IPv6 mandatory) |

# 핵심 개념
- **비연결형**: 세션 설정(Handshake)이 없어 오버헤드가 낮지만, 혼잡 제어나 재전송은 애플리케이션이 직접 처리해야 합니다.
- **데이터그램 단위**: 메시지 단위가 보존되며, 순서 보장 없음. 손실 시 상위 계층에서 보완 필요.
- **멀티플렉싱**: 포트 번호를 통해 여러 애플리케이션이 동일 IP에서 동시에 통신.
- **체크섬**: 의사 헤더 포함해 전송층/네트워크층 필드를 함께 검증함으로써 잘못된 경로의 패킷도 탐지 가능.

# 실무/시험 포인트
- DNS, DHCP, SNMP, RTP 등은 UDP를 사용하며, 방화벽/ACL에서 허용 포트를 명확히 지정해야 합니다.
- UDP 기반 프로토콜은 혼잡 제어가 없으므로, QoS나 트래픽 셰이핑으로 네트워크 보호가 필요합니다.
- VoIP 품질 문제 분석 시 지연·지터·패킷 손실이 핵심 지표이며, UDP 패킷 캡처로 RTP 시퀀스 증가를 확인합니다.
- 시험에서는 TCP와의 비교(연결 설정 여부, 신뢰성, 헤더 크기 등)와 UDP 사용 사례를 자주 묻습니다.
