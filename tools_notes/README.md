# 개요
`tools_notes/`는 운영체제·보안·개발 도구 사용법을 빠르게 참조하기 위한 요약 모음입니다. 모든 문서는 명령 위주로 정리하고, 세부 옵션 표와 예제는 별도 섹션에 배치합니다.

## TL;DR
- `대표 문서 빠른 링크`에서 바로 필요한 도구를 찾고, 문서 TL;DR과 주의 사항을 먼저 확인하세요.
- 명령은 그대로 실행하기 전에 환경 변수·경로를 점검하고, 결과 로그를 TODO 섹션에 기록합니다.
- 공통 옵션은 `docs/writing_guidelines.md`를 참고해 중복 설명을 줄이고, 변경 시 README 표를 함께 갱신합니다.

## 사용 가이드
- 문서 서두에 명령 개요와 주의 사항을 기술합니다.
- 스크립트 및 설정 파일은 `code/` 또는 `examples/` 하위 디렉터리에 보관합니다.
- 키보드 단축키 문서는 플랫폼별 차이를 명확히 표기합니다.

> **TIP**: 도구별 공통 옵션(예: `docker run -v`, `nmap -sC`)은 요약 표로 묶고 세부 설명은 각 문서의 `핵심 개념` 섹션에 링크하세요.

## 하위 주제
- `docker/`, `kubernetes/`: 컨테이너 운영 명령 모음.
- `firewall/`, `scan/`, `sniffing/`: 네트워크 보안/탐지 도구.
- `git/`, `gcc/`: 개발 도구 활용법.
- `monitoring/`, `ELK/`: 로그 수집과 가시화 도구.
- `terminal/`, `linux_key.md`: 셸 및 터미널 단축키 정리.
